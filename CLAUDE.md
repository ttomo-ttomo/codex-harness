# CLAUDE.md

プロジェクト全体のルールは `@AGENTS.md` を参照すること。
このファイルには **Claude Code 固有** の指示のみを記す。

## Claude Code 固有の運用

### サブエージェントの使い分け

- `architect` — 新機能・リファクタリングの要件と計画を起草するときに呼ぶ
- `critic` — architect の成果物を承認ゲート前に批評させるときに呼ぶ（`/feature` から自動）
- `triage` — Codex の出力を分類するときに呼ぶ（必須/推奨/ノイズ）。段階 3 では 4 視点モード

### Codex MCP の呼び出し

`.mcp.json` で Codex MCP サーバーを登録済み。
段階 2 以降では以下のパターンで呼び出す：

- レビュー依頼: `mcp__codex__review` にタスクノートと差分を渡す
- 実装依頼: `mcp__codex__implement`（段階 3 のみ）

### コマンド選択の基準

ユーザーから依頼が来たら、まず規模を見極める：

1. **曖昧なら聞き返す**。「これは `/fix` で良いですか？ それとも `/task`？」
2. **アーキテクチャに触れるなら `/feature`**。ADR が必要な変更は必ずこちら
3. **不明瞭な依頼は `/task` にフォールバック**。task-note を起草すること自体が要件整理になる

### ワーキングディレクトリ

- `.claude/work/<task-id>/` — 段階 2・3 の作業ファイル置き場
- task-id は `YYYYMMDD-<短い要約>` 形式

### 禁止事項

- **ユーザーの明示承認なしに `/feature` の実装フェーズに入らない**
- **triage の「ノイズ」分類を独断で「推奨」に昇格させない**（人間に判断を委ねる）
- **task-note なしで段階 2 以降を進めない**
