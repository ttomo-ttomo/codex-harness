# /fix — 段階 1: Quick

**適用範囲**: 1 ファイル内の変更、タイポ、ロジックの小修正、既存パターンの踏襲

## 手順

1. **スコープ確認**
   - `$ARGUMENTS` の内容が段階 1 の適用範囲に収まるか判定
   - 複数ファイルに波及しそうなら「これは `/task` のほうが適切です」とユーザーに提案して停止

2. **実装**
   - AGENTS.md の「コーディング規約」と「セキュリティ」を遵守
   - 関連する既存ファイルを読んでからコードを書く
   - テストが既にあれば更新、なければ追加可否を判断

3. **セルフチェック**
   - lint + typecheck + test をローカル実行
   - 失敗したら修正して再実行（最大 3 回）
   - 3 回で解決しなければ `/task` へエスカレーション

4. **Codex 軽レビュー**
   - `.claude/skills/codex-review/SKILL.md` の手順に従い、`codex exec` で差分レビューを実行
   - 多ターン対話は行わない（一発レビューのみ）

5. **triage**
   - `triage` サブエージェントに Codex 出力を渡し、必須/推奨/ノイズで分類
   - **必須のみ適用**。推奨はユーザーに判断を委ねる

6. **コミット提案**
   - 差分サマリと適用した triage 結果を提示
   - ユーザーの明示承認後にコミット

## エスカレーション条件

以下に該当したら段階 2 へ：

- 3 ファイル以上に変更が波及
- テストを新規作成する必要がある
- 既存 ADR に該当しない設計判断が必要
- Codex が 5 件以上の必須指摘を出した
